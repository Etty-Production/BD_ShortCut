## Конспект главы 6: Язык запросов SQL

### Что такое SQL

**SQL (Structured Query Language)** — структурированный язык запросов для реляционных БД.

**Основа:** реляционное исчисление кортежей (не реляционная алгебра).

**Тип языка:** **непроцедурный (декларативный)** — пользователь указывает *что* получить, а не *как*.

**Важно:** SQL автономно не используется, он погружён в среду языка программирования СУБД (встроенный SQL).

**Стандарты:** SQL-86, SQL-89, SQL-92, SQL-99, SQL-2003.

**Процедурные расширения:**
- **PL/SQL** — для Oracle.
- **Transact-SQL** — для Microsoft SQL Server.

---

### Классификация операторов SQL

| Подъязык | Назначение | Основные операторы |
|---|---|---|
| **DDL** (Data Definition Language) | Определение структуры данных | `CREATE`, `ALTER`, `DROP` |
| **DML** (Data Manipulation Language) | Манипулирование данными | `SELECT`, `INSERT`, `UPDATE`, `DELETE` |
| **DCL** (Data Control Language) | Управление доступом | `GRANT`, `REVOKE`, `DENY` |
| **TCL** (Transaction Control Language) | Управление транзакциями | `BEGIN TRANSACTION`, `COMMIT`, `ROLLBACK`, `SAVE TRANSACTION` |

## 6.1. Запросы манипулирования данными (DML)

### 6.1.1. Запросы выборки — SELECT

**Полный синтаксис SELECT (упрощённо):**

```sql
SELECT [ALL | DISTINCT | DISTINCTROW | TOP n] <список полей или выражений>
FROM <список таблиц>
[INNER | LEFT | RIGHT JOIN ... ON ...]
[WHERE <условие отбора>]
[GROUP BY <список полей>]
[HAVING <условие отбора>]
[ORDER BY <поле> [ASC | DESC]]
```

---

#### Простейший SELECT

**Пример 1 (одна таблица):**
```sql
SELECT Фамилия, Имя, Отчество, [год рождения] FROM Клиенты;
```

**Пример 2 (все поля):**
```sql
SELECT * FROM Клиенты;
```

**Пример 3 (вычисления без таблицы):**
```sql
SELECT 5+5;
SELECT 5*exp(-5);
```

---

#### Вычисляемые поля и псевдонимы (AS)

**Пример (одна таблица):**
```sql
SELECT 
    [Фамилия] & " " & [Имя] & " " & [Отчество] AS Полное_имя,
    DateDiff("yyyy", [Дата Рождения], Date()) AS Возраст
FROM Клиенты;
```

**Для SQL Server (конкатенация через +, GETDATE() вместо Date()):**
```sql
SELECT 
    [Фамилия] + ' ' + [Имя] + ' ' + [Отчество] AS Полное_имя,
    DATEDIFF(year, [Дата рождения], GETDATE()) AS Возраст
FROM Клиенты;
```

---

#### Управление повторяющимися строками

| Предикат | Действие |
|---|---|
| `ALL` (по умолчанию) | Вывод всех строк, включая повторы |
| `DISTINCT` | Исключает строки с повторяющимися значениями в выбранных полях |
| `DISTINCTROW` (Access) | Исключает полностью повторяющиеся записи (для многотабличных запросов) |
| `TOP n` | Возвращает первые n записей (обычно с `ORDER BY`) |

**Пример (одна таблица):**
```sql
SELECT DISTINCT Город FROM Клиенты;
```

```sql
SELECT TOP 5 Фамилия, Имя FROM Клиенты ORDER BY Фамилия;
```

```sql
SELECT TOP 10 PERCENT Фамилия, Цена FROM Товары ORDER BY Цена DESC;
```

---

#### Фраза WHERE — условия отбора

**Операторы сравнения:** `=`, `<>` (или `!=`), `<`, `>`, `<=`, `>=`

**Логические операторы:** `NOT`, `AND`, `OR`, `XOR`

**Примеры (одна таблица):**
```sql
SELECT Фамилия FROM Клиенты WHERE Город = 'Минск';
```

```sql
SELECT * FROM Клиенты WHERE Отчество IS NULL;
```

```sql
SELECT * FROM Клиенты WHERE Возраст BETWEEN 18 AND 30;
```

```sql
SELECT * FROM Клиенты WHERE Город IN ('Минск', 'Москва', 'Киев');
```

---

#### LIKE — поиск по шаблону

| Символ (Access) | Символ (SQL Server) | Значение |
|---|---|---|
| `*` | `%` | Любое количество любых символов |
| `?` | `_` | Один любой символ |
| `#` | — | Одна любая цифра (только Access) |
| `[А-Я]` | `[А-Я]` | Один символ из диапазона |
| `[!А-Я]` | `[^А-Я]` | Один символ не из диапазона |

**Примеры (одна таблица, Access):**
```sql
-- Фамилии, начинающиеся на А
SELECT * FROM Клиенты WHERE Фамилия LIKE "А*";

-- Почтовый индекс, начинающийся с 220
SELECT * FROM Клиенты WHERE Индекс LIKE "220###";
```

**Примеры (SQL Server):**
```sql
SELECT * FROM Товары WHERE Наименование LIKE '%чай%';
SELECT * FROM Клиенты WHERE Паспорт LIKE 'MP223[34][4-6]_';
```

#### JOIN — соединение таблиц

**Типы соединений:**

| Тип | Ключевое слово | Что возвращает |
|---|---|---|
| Внутреннее (естественное) | `INNER JOIN` | Только записи, имеющие совпадения в обеих таблицах |
| Левое внешнее | `LEFT JOIN` | Все записи из левой таблицы + совпадающие из правой (несовпавшие — NULL) |
| Правое внешнее | `RIGHT JOIN` | Все записи из правой таблицы + совпадающие из левой |
| Полное внешнее | `FULL OUTER JOIN` | Все записи из обеих таблиц |

**Пример (две таблицы, но для понимания):**
```sql
-- Только клиенты, у которых есть заказы
SELECT Клиенты.Фамилия, Заказы.Дата, Заказы.Цена
FROM Клиенты 
INNER JOIN Заказы ON Клиенты.Код_клиента = Заказы.Код_клиента;
```

```sql
-- Все клиенты (даже без заказов)
SELECT Клиенты.Фамилия, Заказы.Дата, Заказы.Цена
FROM Клиенты 
LEFT JOIN Заказы ON Клиенты.Код_клиента = Заказы.Код_клиента;
```

**Альтернативный способ соединения (декартово произведение + WHERE):**
```sql
SELECT Клиенты.Фамилия, Заказы.Дата
FROM Клиенты, Заказы
WHERE Клиенты.Код_клиента = Заказы.Код_клиента;
```

> **Важно:** первый способ (JOIN) предпочтительнее — он даёт более оптимальный план выполнения.

---

#### Унарная связь (самосоединение)

Когда таблица ссылается сама на себя (поле «Подчиняется» ссылается на первичный ключ «Код_сотрудника»).

**Пример:**
```sql
SELECT 
    Сотр.Фамилия, 
    Сотр.Должность, 
    Сотр1.Фамилия AS Начальник
FROM Сотрудники AS Сотр
LEFT JOIN Сотрудники AS Сотр1 ON Сотр.Подчиняется = Сотр1.КодСотрудника;
```

### 6.1.2. Запросы с группировкой (GROUP BY)

**Назначение:** получение итоговых значений по группам (сумма, среднее, количество и т.д.).

**Агрегатные (групповые) функции:**

| Функция | Действие |
|---|---|
| `SUM(поле)` | Сумма значений |
| `AVG(поле)` | Среднее арифметическое |
| `COUNT(поле)` | Количество непустых значений |
| `COUNT(*)` | Количество записей |
| `MIN(поле)` | Минимальное значение |
| `MAX(поле)` | Максимальное значение |
| `STDEV(поле)` | Стандартное отклонение |
| `VAR(поле)` | Дисперсия |
| `FIRST(поле)` | Первое значение |
| `LAST(поле)` | Последнее значение |

**Важное правило:** Во фразе `SELECT` при использовании `GROUP BY` могут быть только:
1. Поля, перечисленные в `GROUP BY`.
2. Агрегатные функции.
3. Константы.
4. Выражения, включающие только эти элементы.

**Пример (одна таблица):**
```sql
-- Количество клиентов в каждом городе
SELECT Город, COUNT(*) AS Количество
FROM Клиенты
GROUP BY Город;
```

```sql
-- Средняя цена товаров по категориям
SELECT Категория, AVG(Цена) AS Средняя_цена
FROM Товары
GROUP BY Категория;
```

```sql
-- Товары, заказываемые более чем одним покупателем (требует две таблицы)
SELECT Код_товара, COUNT(*) AS Кол_во_заказов
FROM Заказы
GROUP BY Код_товара
HAVING COUNT(*) > 1;
```

---

#### Фраза HAVING

**Отличие HAVING от WHERE:**

| Фраза | Применяется | К чему относится |
|---|---|---|
| `WHERE` | До группировки | К отдельным строкам (не может содержать агрегатные функции) |
| `HAVING` | После группировки | К группам (может содержать агрегатные функции) |

**Пример:**
```sql
-- Города, где живут более 3 клиентов
SELECT Город, COUNT(*) AS Количество
FROM Клиенты
GROUP BY Город
HAVING COUNT(*) > 3;
```

---

#### Фраза ORDER BY — сортировка

```sql
SELECT * FROM Товары ORDER BY Цена DESC;
SELECT Фамилия, Имя FROM Клиенты ORDER BY Фамилия ASC, Имя ASC;
```

**Важно:** `TOP` всегда используется с `ORDER BY`.

```sql
SELECT TOP 10 Наименование, Цена FROM Товары ORDER BY Цена DESC;
```

### 6.1.3. Параметрические (динамические) запросы

Условие отбора задаётся параметром, который вводится во время выполнения.

**Access:**
```sql
PARAMETERS [Введите год:] INT;
SELECT Фамилия FROM Клиенты 
WHERE Year(Дата_рождения) = [Введите год:];
```

**SQL Server:**
```sql
DECLARE @p_year INT;
SET @p_year = 2005;
SELECT Фамилия FROM Клиенты 
WHERE YEAR(Дата_рождения) = @p_year;
```

---

### 6.1.4. Вложенные запросы (подзапросы)

`SELECT` может быть вложен в другой `SELECT` (обычно во фразу `WHERE`).

**Пример (одна таблица):**
```sql
-- Клиенты с кредитом ниже среднего
SELECT Фамилия, Имя, Кредит
FROM Клиенты
WHERE Кредит < (SELECT AVG(Кредит) FROM Клиенты);
```

---

#### Предикаты для работы с подзапросами

| Предикат | Значение |
|---|---|
| `EXISTS` | Истина, если подзапрос вернул хотя бы одну строку |
| `IN` | Истина, если значение есть в результате подзапроса |
| `SOME` / `ANY` | Истина, если сравнение верно хотя бы для одной строки подзапроса |
| `ALL` | Истина, если сравнение верно для всех строк подзапроса |

**Пример с EXISTS (две таблицы):**
```sql
-- Клиенты, купившие компьютер Pentium II 350
SELECT Фамилия FROM Клиенты
WHERE EXISTS (
    SELECT * FROM Заказы 
    WHERE Заказы.код_клиента = Клиенты.код_клиента 
    AND [Наименование товара] = 'Pentium II 350'
);
```

**Пример с IN (одна таблица):**
```sql
-- Товары с ценой, совпадающей с ценой любого товара, проданного со скидкой (требует две таблицы)
SELECT Наименование FROM Товары
WHERE Цена IN (SELECT Цена FROM Заказы WHERE Скидка >= 0.25);
```

**Пример с ANY (одна таблица):**
```sql
-- Товары дороже самого дешёвого товара со скидкой 25%
SELECT Наименование FROM Товары
WHERE Цена > ANY (SELECT Цена FROM Заказы WHERE Скидка >= 0.25);
```

**Пример с ALL (одна таблица):**
```sql
-- Товары дороже самого дорогого товара со скидкой 25%
SELECT Наименование FROM Товары
WHERE Цена > ALL (SELECT Цена FROM Заказы WHERE Скидка >= 0.25);
```

## 6.2. Запросы действий (модификации данных)

### 6.2.1. Запрос на обновление (UPDATE)

```sql
UPDATE <таблица> 
SET <поле> = <выражение> [, <поле> = <выражение>]
[WHERE <условие>]
```

**Пример (одна таблица):**
```sql
UPDATE Товары 
SET Наименование = 'Pentium III 800', Цена = Цена + 250
WHERE Наименование = 'Pentium II 350';
```

---

### 6.2.2. Запрос на добавление (INSERT INTO)

**Форма 1 — добавление одной записи с явными значениями:**
```sql
INSERT INTO <таблица> [(<список полей>)] 
VALUES (<список значений>)
```

**Пример (одна таблица):**
```sql
INSERT INTO Товары (Наименование, Цена) 
VALUES ('Pentium II 233', 450);
```

**Форма 2 — добавление записей из результата SELECT:**
```sql
INSERT INTO <таблица> [(<список полей>)] 
<инструкция SELECT>
```

**Пример (одна таблица, но копирование в ту же таблицу требует осторожности):**
```sql
-- Копирование клиентов из одного города в архивную таблицу
INSERT INTO Клиенты_Архив (Фамилия, Имя, Отчество)
SELECT Фамилия, Имя, Отчество FROM Клиенты 
WHERE Город = 'Минск';
```

---

### 6.2.3. Запрос на удаление (DELETE)

```sql
DELETE FROM <таблица> [WHERE <условие>]
```

**Пример (одна таблица):**
```sql
DELETE FROM Товары WHERE Цена < 100;
```

> **Внимание:** `DELETE FROM Товары;` удалит все записи из таблицы.

---

### 6.2.4. Запрос на создание таблицы (SELECT INTO)

```sql
SELECT <список полей> INTO <новая_таблица>
FROM <таблица> [WHERE <условие>]
```

**Пример (одна таблица):**
```sql
SELECT Фамилия, Имя, Отчество INTO Совершеннолетние
FROM Клиенты 
WHERE [год рождения] < 2006;
```

## 6.3. Специальные запросы

### 6.3.1. Перекрёстные запросы (TRANSFORM) — только Access

Создают сводные таблицы (строки — один параметр, столбцы — другой, ячейки — агрегатное значение).

```sql
TRANSFORM <агрегатная функция>
SELECT <поля для строк> [GROUP BY]
PIVOT <поля для столбцов>
```

**Пример:**
```sql
TRANSFORM Sum(Заказы.Цена * Заказы.Количество) AS СуммаПоТовару
SELECT Товары.Товар
FROM Товары INNER JOIN Заказы ON Товары.КодТовара = Заказы.КодТовара
GROUP BY Товары.Товар
PIVOT Format([Дата заказа], "mmm yyyy");
```

---

### 6.3.2. Запрос на объединение (UNION)

Объединяет результаты нескольких `SELECT` по вертикали.

```sql
<SELECT_1>
UNION
<SELECT_2>
[ORDER BY ...]
```

**Пример:**
```sql
SELECT Фамилия & " " & Имя & " " & Отчество AS Название, Город 
FROM Клиенты
UNION
SELECT Поставщик, Город FROM Поставщики
ORDER BY Название;
```

**Другие операции (стандарт SQL2):**
- `EXCEPT` — разность (записи из первого, отсутствующие во втором)
- `INTERSECT` — пересечение (записи, присутствующие в обоих)

## 6.4. Запросы определения данных (DDL)

### 6.4.1. Создание представлений (VIEW)

**Представление** — виртуальная таблица, построенная на основе запроса выборки. Само не хранит данные, а извлекает их при обращении.

```sql
CREATE VIEW <имя> [(<список полей>)] AS <SELECT>
```

**Пример:**
```sql
CREATE VIEW Заказы_клиентов (клиент, товар, цена, дата) AS
SELECT Фамилия + ' ' + Имя + ' ' + Отчество, Товар, Цена, Дата
FROM Клиенты INNER JOIN Заказы ON Клиенты.код_клиента = Заказы.код_клиента
WHERE Город = 'Минск';
```

**Удаление представления:**
```sql
DROP VIEW <имя>
```

**Преимущества представлений:**

| Преимущество | Пояснение |
|---|---|
| Независимость от данных | Интерфейс представления не меняется при изменении базовых таблиц (логическая независимость) |
| Повышение защищённости | Можно дать доступ только к определённым представлениям, скрывая остальные данные |
| Снижение сложности | Сложные запросы инкапсулируются в представлении |
| Индивидуальная настройка | Каждый пользователь видит только нужные ему данные в удобной форме |

**Недостатки:** снижение производительности, структурные ограничения, ограниченные возможности обновления.

---

### 6.4.2. Создание доменов (CREATE TYPE — SQL Server)

```sql
CREATE TYPE <имя> FROM <тип данных> [NOT NULL]
```

**Пример:**
```sql
CREATE TYPE sex FROM char(1) NOT NULL;
```

**Удаление домена:**
```sql
DROP TYPE <имя> [RESTRICT | CASCADE]
```

---

### Типы данных SQL Server (основные)

| Категория | Типы |
|---|---|
| **Строковые** | `char(n)`, `varchar(n)`, `nchar(n)`, `nvarchar(n)`, `text`, `ntext` |
| **Числовые целые** | `bit`, `tinyint`, `smallint`, `int`, `bigint` |
| **Числовые точные** | `decimal(p,s)`, `numeric(p,s)`, `money`, `smallmoney` |
| **Числовые приблизительные** | `float(n)`, `real` |
| **Дата/время** | `date`, `time(n)`, `smalldatetime`, `datetime` |
| **Бинарные** | `binary(n)`, `varbinary(n)`, `image` |

### 6.4.3. Создание таблиц (CREATE TABLE)

**Простая форма (Access):**
```sql
CREATE TABLE Товары (
    Код CHAR(5) NOT NULL,
    Тип VARCHAR(20),
    Наименование VARCHAR(200) NOT NULL,
    Цена DECIMAL(8,2)
);
```

**Расширенная форма (SQL Server) с ограничениями:**
```sql
CREATE TABLE Товары (
    Код_товара CHAR(5) NOT NULL PRIMARY KEY,
    Наименование VARCHAR(50) NOT NULL,
    Цена FLOAT CHECK (Цена >= 0)
);
```

```sql
CREATE TABLE Заказы (
    КодЗаказа INT IDENTITY(1,1) NOT NULL PRIMARY KEY,
    КодКлиента INT,
    ДатаЗаказа DATETIME DEFAULT GETDATE(),
    ОтпускнаяЦена SMALLMONEY DEFAULT 0 CHECK (ОтпускнаяЦена BETWEEN 0 AND 100000),
    CONSTRAINT FK_Заказы_Клиенты FOREIGN KEY (КодКлиента) 
        REFERENCES Клиенты(Код_клиента)
        ON UPDATE CASCADE 
        ON DELETE CASCADE
);
```

**Действия для внешних ключей:**

| Действие | Эффект |
|---|---|
| `CASCADE` | Каскадное обновление/удаление |
| `SET NULL` | Внешний ключ устанавливается в NULL |
| `SET DEFAULT` | Внешний ключ устанавливается в значение по умолчанию |
| `NO ACTION` | Запрет (используется по умолчанию) |

---

### 6.4.4. Создание и удаление индексов

```sql
CREATE [UNIQUE] INDEX <имя> ON <таблица> (<поле> [ASC|DESC] [, ...])
```

**Примеры (одна таблица):**
```sql
CREATE INDEX AscName ON Клиенты (Фамилия, Имя, Отчество);
CREATE UNIQUE INDEX passportasc ON Клиенты (Паспорт);
```

**Удаление индекса:**
```sql
DROP INDEX <имя>
```

## 6.5. Использование транзакций

**Транзакция** — логическая единица работы, которая либо выполняется полностью, либо полностью откатывается.

**Управление транзакциями:**

| Оператор | Действие |
|---|---|
| `BEGIN TRANSACTION` | Начало транзакции |
| `COMMIT` | Фиксация всех изменений (успешное завершение) |
| `ROLLBACK` | Откат всех изменений в транзакции |
| `SAVE TRANSACTION` | Установка точки сохранения |

**Уровни изоляции транзакций (`SET TRANSACTION ISOLATION LEVEL`):**

| Уровень | Описание |
|---|---|
| `READ UNCOMMITTED` | Незавершённое считывание (грязное чтение) |
| `READ COMMITTED` | Завершённое считывание |
| `REPEATABLE READ` | Повторяемое считывание |
| `SERIALIZABLE` | Полная безопасность (сериальный план выполнения) |

## 6.6. Управление доступом к данным (DCL)

**Дискреционная модель разграничения доступа** — каждому пользователю назначаются привилегии на объекты БД.

**Предоставление прав (GRANT):**
```sql
GRANT SELECT, INSERT ON Клиенты TO Иванов;
GRANT ALL PRIVILEGES ON Клиенты TO Петров WITH GRANT OPTION;
GRANT SELECT ON Клиенты TO PUBLIC;
```

**Отзыв прав (REVOKE):**
```sql
REVOKE SELECT, INSERT ON Клиенты FROM Иванов;
REVOKE GRANT OPTION FOR SELECT ON Клиенты FROM Петров CASCADE;
```

**Набор привилегий:**

| Привилегия | Действие |
|---|---|
| `SELECT` | Просмотр данных |
| `INSERT` | Добавление данных |
| `UPDATE` | Обновление данных |
| `DELETE` | Удаление данных |
| `REFERENCES` | Ссылаться на поле при определении ссылочной целостности |
| `USAGE` | Использовать домены и ограничители |

**Опции:**
- `WITH GRANT OPTION` — пользователь может передавать права другим.
- `PUBLIC` — права предоставляются всем пользователям.
- `CASCADE` — отзыв прав распространяется на тех, кто получил права через `GRANT OPTION`.

## Ответы на контрольные вопросы

### Вопрос 1. К какому виду реляционных языков относится SQL?
**Ответ:** SQL относится к **реляционному исчислению** (непроцедурный, декларативный язык), а не к реляционной алгебре (которая процедурна).

### Вопрос 2. На какие группы можно разбить основные операторы SQL?
- **DDL** — язык определения данных (`CREATE`, `ALTER`, `DROP`).
- **DML** — язык манипулирования данными (`SELECT`, `INSERT`, `UPDATE`, `DELETE`).
- **DCL** — язык управления доступом (`GRANT`, `REVOKE`, `DENY`).
- **TCL** — язык управления транзакциями (`BEGIN TRANSACTION`, `COMMIT`, `ROLLBACK`, `SAVE TRANSACTION`).

### Вопрос 3. Перечислите операторы языка описания запросов выборки в SQL.
`SELECT`, `FROM`, `WHERE`, `GROUP BY`, `HAVING`, `ORDER BY`, `JOIN` (INNER, LEFT, RIGHT), `UNION`, `INTERSECT`, `EXCEPT`.

### Вопрос 4. Какие виды вычислений можно производить в SQL?
- Арифметические (`+`, `-`, `*`, `/`, `%`, `^`, `MOD`, `\`).
- Логические (`AND`, `OR`, `NOT`, `XOR`).
- Сравнения (`=`, `<>`, `<`, `>`, `<=`, `>=`).
- Конкатенация (`&` в Access, `+` в SQL Server, `CONCAT`).
- Встроенные функции (`SUM`, `AVG`, `COUNT`, `MIN`, `MAX`, `YEAR`, `DATEADD`, `DATEDIFF`, `IIF`/`CASE`, `ISNULL`).
- Побитовые (`&`, `|`, `^`).

### Вопрос 5. Исправьте ошибку(и) в запросе:
```sql
SELECT Фамилия, Имя, Отчество 
FROM Клиенты 
INNER JOIN Заказы AS Клиенты = Заказы 
SORT BY Фамилия 
WHERE Город = Минск;
```

**Исправленный вариант:**
```sql
SELECT Клиенты.Фамилия, Клиенты.Имя, Клиенты.Отчество
FROM Клиенты 
INNER JOIN Заказы ON Клиенты.КодКлиента = Заказы.КодКлиента
WHERE Клиенты.Город = 'Минск'
ORDER BY Клиенты.Фамилия;
```

**Ошибки:**
1. `AS Клиенты = Заказы` — неверный синтаксис JOIN. Нужно `ON <условие>`.
2. `SORT BY` — нет такого оператора, нужен `ORDER BY`.
3. `Город = Минск` — строковое значение должно быть в кавычках.
4. Имена полей неоднозначны — нужно указать таблицу.

### Вопрос 6. Что сделает следующий запрос (каскадное удаление не установлено):
```sql
DELETE * FROM Клиенты INNER JOIN Заказы ON Клиенты.КодКлиента = Заказы.КодКлиента;
```

**Ответ:** **a) удалит из таблицы «Клиенты» информацию о тех клиентах, которые имеют заказы** (и только их, т.к. INNER JOIN выбирает только клиентов с заказами). Заказы не удалятся, так как каскадное удаление не установлено. Возникнет ошибка ссылочной целостности, если есть связанные записи.

### Вопрос 7. Чем отличается внутреннее соединение от внешнего?

| Тип | Что возвращает |
|---|---|
| **INNER JOIN** | Только записи, имеющие совпадения в обеих таблицах |
| **LEFT JOIN** | Все записи из левой таблицы + совпадающие из правой (несовпавшие — NULL) |
| **RIGHT JOIN** | Все записи из правой таблицы + совпадающие из левой |

**Пример с INNER JOIN (только клиенты с заказами):**
```sql
SELECT Клиенты.Фамилия, Заказы.Дата
FROM Клиенты 
INNER JOIN Заказы ON Клиенты.КодКлиента = Заказы.КодКлиента;
```

**Пример с LEFT JOIN (все клиенты, даже без заказов):**
```sql
SELECT Клиенты.Фамилия, Заказы.Дата
FROM Клиенты 
LEFT JOIN Заказы ON Клиенты.КодКлиента = Заказы.КодКлиента;
```

### Вопрос 8. Что позволяют запросы с группировкой?
Получать **итоговые значения по группам** (сумму, среднее, количество, минимум, максимум и т.д.). Пример: общая сумма продаж по каждому клиенту, средняя цена товаров по категориям.

### Вопрос 9. Какой из запросов действия имеет две формы записи?
**INSERT INTO** — имеет две формы:
1. Добавление одной записи с явными значениями (`VALUES`).
2. Добавление записей, отобранных из другой таблицы (`SELECT`).

### Вопрос 10. Напишите запрос выборки фамилии, имени и отчества всех клиентов, которые покупали товары в текущем году на языке SQL. Записи должны быть упорядочены по возрастанию по полю «Фамилия». Повторяющиеся записи должны быть исключены.

```sql
SELECT DISTINCT Клиенты.Фамилия, Клиенты.Имя, Клиенты.Отчество
FROM Клиенты 
INNER JOIN Заказы ON Клиенты.КодКлиента = Заказы.КодКлиента
WHERE YEAR(Заказы.Дата) = YEAR(GETDATE())
ORDER BY Клиенты.Фамилия ASC;
```

### Вопрос 11. Что возвратит следующий запрос?
```sql
SELECT КодКлиента, SUM(Цена*Количество) 
FROM Клиенты 
INNER JOIN Заказы ON Клиенты.КодКлиента = Заказы.КодКлиента
WHERE EXISTS (
    SELECT * FROM Товары 
    WHERE НаименованиеТовара = 'Молоко' 
    AND Товары.КодТовара = Заказы.КодТовара
)
GROUP BY КодКлиента;
```

**Ответ:** Для каждого клиента, который покупал **молоко**, запрос вернёт **общую сумму всех его заказов** (не только молока), потому что `SUM` не ограничена условием `EXISTS`. Если нужна только сумма за молоко, условие должно быть во `FROM` или в `WHERE` без `EXISTS`.

### Вопрос 12. Что возвратит следующий запрос (связи между таблицами заданы):
```sql
SELECT * FROM Клиенты, Заказы;
```

**Ответ:** **c) любые возможные сочетания записей из таблицы «Клиенты» и записей из таблицы «Заказы»** (декартово произведение). Если в «Клиенты» 10 записей, а в «Заказы» 100, результат будет 1000 записей.

### Вопрос 13. Перечислите основные операторы языка определения данных в SQL.
`CREATE TABLE`, `ALTER TABLE`, `DROP TABLE`, `CREATE INDEX`, `DROP INDEX`, `CREATE VIEW`, `ALTER VIEW`, `DROP VIEW`, `CREATE TYPE`, `ALTER TYPE`, `DROP TYPE`.

### Вопрос 14. Какие типы данных поддерживаются в SQL?
Строковые (`CHAR`, `VARCHAR`, `NCHAR`, `NVARCHAR`, `TEXT`), целочисленные (`INT`, `SMALLINT`, `TINYINT`, `BIGINT`, `BIT`), точные числовые (`DECIMAL`, `NUMERIC`, `MONEY`), приблизительные (`FLOAT`, `REAL`), дата/время (`DATE`, `TIME`, `DATETIME`, `SMALLDATETIME`), бинарные (`BINARY`, `VARBINARY`, `IMAGE`).

### Вопрос 17. Какие объекты базы данных позволяют создавать операторы языка определения данных в SQL?
- Таблицы (`CREATE TABLE`)

### Вопрос 18. Как организуется защита данных от неавторизованного доступа в SQL?
С помощью операторов **GRANT** (выдача прав), **REVOKE** (отзыв прав) и **DENY** (запрет). Реализуется **дискреционная модель разграничения доступа** на основе идентификаторов пользователей и назначенных им привилегий (`SELECT`, `INSERT`, `UPDATE`, `DELETE`, `REFERENCES`, `USAGE`). Владелец объекта имеет полный доступ и может передавать права другим.

## Все примеры SQL (на одной таблице) — сводка

```sql
-- Простая выборка
SELECT * FROM Клиенты;

-- Выборка с условием
SELECT Фамилия, Имя FROM Клиенты WHERE Город = 'Минск';

-- Уникальные значения
SELECT DISTINCT Город FROM Клиенты;

-- Сортировка
SELECT Фамилия, Имя FROM Клиенты ORDER BY Фамилия;

-- TOP (первые 5)
SELECT TOP 5 Фамилия, Имя FROM Клиенты ORDER BY Фамилия;

-- LIKE
SELECT * FROM Клиенты WHERE Фамилия LIKE 'И%';

-- Агрегатная функция без группировки
SELECT AVG(Кредит) FROM Клиенты;

-- Группировка
SELECT Город, COUNT(*) FROM Клиенты GROUP BY Город;

-- HAVING
SELECT Город, COUNT(*) FROM Клиенты GROUP BY Город HAVING COUNT(*) > 3;

-- Подзапрос (среднее)
SELECT Фамилия, Кредит FROM Клиенты 
WHERE Кредит > (SELECT AVG(Кредит) FROM Клиенты);

-- UPDATE
UPDATE Клиенты SET Город = 'Минск' WHERE Город = 'Минск ';

-- INSERT (одна запись)
INSERT INTO Клиенты (Фамилия, Имя, Город) VALUES ('Иванов', 'Иван', 'Москва');

-- DELETE
DELETE FROM Клиенты WHERE Код_клиента = 5;

-- SELECT INTO (создание новой таблицы)
SELECT Фамилия, Имя INTO Клиенты_Минск FROM Клиенты WHERE Город = 'Минск';

-- CREATE TABLE
CREATE TABLE Клиенты_Новые (
    Код INT PRIMARY KEY,
    Фамилия VARCHAR(50) NOT NULL
);

-- CREATE INDEX
CREATE INDEX idx_Фамилия ON Клиенты(Фамилия);
```
