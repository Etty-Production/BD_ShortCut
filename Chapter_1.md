## Конспект главы 1: Основные понятия баз данных (БД)

### 1. Что такое информация и данные? (Вводная часть)

Прежде чем говорить о базах данных, авторы вводят два важных понятия:

*   **Информация** — это любые сведения, которые мы получаем из окружающей среды, передаём другим или храним внутри системы (например, в голове или компьютере). Всегда есть **источник** (кто/что даёт) и **приёмник** (кто/что получает).
*   **Данные** — это **записанная** информация в виде, который позволяет автоматизировать её сбор, хранение и обработку компьютером.

**Простыми словами:** Информация — это смысл. Данные — это форма для хранения этого смысла.
*Пример:* Числа «25-05-99» — это данные. А вот смысл зависит от договорённости (интерпретации): это может быть дата (25 мая 1999 г.) или номер телефона (25-05-99). Чтобы понять данные, нужны **соглашения** (интерпретация).

### 2. Определение базы данных (БД)

Автор приводит классическое определение **Дж. Мартина** и своё рабочее определение.

**Определение БД (по Мартину):**
> БД — это совокупность взаимосвязанных данных при такой их минимальной избыточности, которая допускает их использование оптимальным образом для одного или нескольких приложений в определённой предметной области человеческой деятельности.

**Простыми словами:**
БД — это не просто куча файлов, а **аккуратно организованный набор взаимосвязанных данных**, где почти нет повторов, и который удобно использовать для решения конкретных задач (например, список студентов + их оценки).

**Важные уточнения:**
*   БД — это **динамическая модель** части реального мира (предметной области). Идеально отразить весь мир нельзя, всегда идут на упрощения.
*   Часто под БД ошибочно понимают **СУБД** (систему управления БД) — программу, которая управляет базами данных.

### 3. База данных (БД) vs Система управления БД (СУБД)

Это ключевое различие для зачёта.

| | **База данных (БД)** | **СУБД** |
|---|---|---|
| **Что это?** | Сами данные (файлы на диске) | Программа для управления БД |
| **Пример** | Файл `students.db` | Microsoft Access, PostgreSQL, MySQL, Oracle |
| **Состав** | Только взаимосвязанные данные | Ядро + язык запросов + интерфейс + средства администрирования |

**Вывод:** БД — это содержимое. СУБД — это инструмент для работы с содержимым.

### 4. Свойства технологии баз данных (5 ключевых свойств)

Набор данных можно назвать БД, только если он обладает этими свойствами:

1.  **Связность и структурированность:** Данные не хаотичны. В столбце «Фамилия» — только фамилии, в столбце «Имя» — только имена. Есть чёткая структура.
2.  **Контролируемая избыточность:** Каждая логическая единица хранится **один раз**. Повторы допустимы только там, где это ускоряет работу (например, резервные копии), но они контролируются.
3.  **Целостность:** Данные всегда **правильные** и **непротиворечивые**. Если в реальном мире что-то изменилось, в БД тоже должно измениться.
4.  **Физическая и логическая независимость:**
    *   *Физическая:* Есть данные — не важно, на каком диске или в каком файле они лежат. Программы от этого не ломаются.
    *   *Логическая:* Можно добавить новое поле (например, «Отчество») в таблицу «Студенты», и старые программы, которые не используют это поле, продолжат работать.
5.  **Безопасность и секретность:**
    *   *Безопасность:* Защита от взлома, порчи данных.
    *   *Секретность:* Контроль того, кто и какую информацию может увидеть (пароли, права доступа).

### 5. Четыре этапа работы с БД

1.  **Определение структуры** — проектирование таблиц и связей.
2.  **Ввод данных** — заполнение форм.
3.  **Поиск/выборка** — формирование **запроса** (набора условий поиска).
4.  **Оформление** — вывод результата в виде отчёта, таблицы, графика.

**4 основных компонента реляционной БД:** Таблицы, Запросы, Формы, Отчёты.

## Разбор вопросов к главе 1 (Для зачёта)

### Вопрос 1. Дайте определение БД.

**Ответ:** База данных (БД) — это поименованная совокупность структурированных взаимосвязанных данных, относящихся к определённой предметной области, хранящихся на физических носителях и организованных таким образом, чтобы обеспечить минимальную избыточность, целостность и независимость данных от прикладных программ.

*Или кратко:* БД — это набор связанных данных, хранящихся по строгим правилам.

### Вопрос 2. Перечислите свойства БД.

1.  Связность, логическая упорядоченность и структурированность.
2.  Контролируемая избыточность.
3.  Целостность.
4.  Физическая и логическая независимость.
5.  Безопасность и секретность.

### Вопрос 3. Опишите пять основных компонентов БД.

1.  **Таблицы:** Основные хранилища данных (как листы Excel). Данные лежат в строках (записи) и столбцах (поля).
2.  **Запросы:** Инструменты для поиска и фильтрации данных по заданным критериям. Результат запроса — тоже таблица.
3.  **Формы:** «Оболочки» для удобного ввода, просмотра и редактирования записей (одна запись на экране).
4.  **Отчёты:** Инструменты для красивого вывода данных на печать (с группировками, итогами).
5.  **Программы (модули):** Код, который связывает всё вышеперечисленное в единое работающее приложение (автоматизация действий, кнопки).

### Вопрос 4. Преимущества БД перед файловой системой (3 главных).

1.  **Устранение дублирования и противоречий:** В файловой системе один и тот же телефон может быть записан в 10 файлах (дублирование). В БД — одна запись, которую используют все программы.
2.  **Независимость от данных:** В файловой системе изменили структуру файла — переписывай все программы. В БД структура хранится отдельно (метаданные).
3.  **Мощный поиск и индексация:** В файлах Windows ищешь только по имени/дате. В БД можно искать по любому слову внутри документов, сортировать, фильтровать мгновенно благодаря индексам.

### Вопрос 5. Архитектура ANSI/SPARC и независимость данных.

*Зарисуйте схему:*
1.  **Внутренний уровень** (физическая модель) — *как лежат данные на диске* (куски, сектора, индексы).
2.  **Концептуальный уровень** (логическая модель) — *что за данные* (таблицы, колонки, связи), без привязки к диску.
3.  **Внешний уровень** (представления/интерфейс) — *как видит пользователь* (только нужные столбцы, удобный формат).

**Независимость:**
*   **Логическая** (между уровнем 2 и 3): Можно добавить новую колонку в таблицу (изменить концептуальный уровень), а внешние отчёты (например, старый отчёт без этой колонки) продолжат работать. Программы не зависят от изменений логики.
*   **Физическая** (между уровнем 1 и 2): Переставили файл БД с диска C: на диск D: или сменили тип диска (HDD → SSD). Концептуальная схема не изменилась, программы не заметили.

### Вопрос 6. Основные характеристики БД как способа хранения.

1.  Структурированность при минимальной избыточности.
2.  Наличие метаданных (описания структуры данных).
3.  Независимость программ и данных / программ и операций.
4.  Встроенные средства целостности, безопасности и секретности.
5.  Разные способы представления (формы, отчёты).
6.  Расширенные средства поиска (индексирование).

### Вопрос 7. Обеспечение целостности. Типы ограничителей.

**Целостность** = данные всегда верны, непротиворечивы и соответствуют реальному миру.

**Средства обеспечения:**
1.  **Транзакции** (неделимые операции: или всё сделалось, или ничего).
2.  **Ограничители целостности (constraints)** — правила, которые проверяет СУБД.

**Типы ограничителей:**
1.  **Ограничители значений:** тип данных (число, текст), диапазон (от 18 лет), список допустимых значений (Муж/Жен).
2.  **Ссылочная целостность:** Правила связей между таблицами. Например, нельзя удалить факультет, если в нём ещё есть студенты (или автоматически удалить студентов — каскадное удаление).
3.  **Целостность записи:** Обязательность полей (нельзя создать студента без фамилии) и существование первичного ключа.

### Вопрос 8. Контролируемая избыточность. Проблемы при её нарушении.

**Основное требование:** Каждая логическая единица данных должна храниться **только один раз** (избыточность только там, где это нужно для скорости, но под контролем).

**Проблемы при нарушении (когда данные повторяются):**
1.  **Аномалия модификации:** Изменили адрес клиента в одной записи, а в другой забыли — данные разошлись.
2.  **Аномалия удаления:** Удалили последний купленный товар, и вместе с ним случайно удалилась информация о самом товаре (больше нигде не было ссылок).
3.  **Аномалия добавления:** Нельзя добавить нового поставщика, пока он не поставил хотя бы один товар (неполные данные).
4.  **Лишняя работа:** Тратим время на ввод одного и того же, тратим место на диске.
5.  **Ошибки ввода:** Чем чаще повторяем ввод, тем больше шанс опечататься.

## Термины для запоминания (Словарь к зачёту)

*   **Данные (Data):** Записанная информация.
*   **Метаданные (Metadata):** Данные о данных (структура таблиц, типы полей).
*   **Предметная область:** Часть реального мира, которую моделирует БД.
*   **Избыточность (Redundancy):** Повторение данных.
*   **Целостность (Integrity):** Правильность и непротиворечивость данных.
*   **Независимость данных (Data Independence):** Защита программ от изменений в структуре или месте хранения данных.
*   **Транзакция (Transaction):** Неделимая группа операций (или всё, или ничего).
*   **Ограничитель целостности (Constraint):** Правило, которому должны следовать данные.
*   **Запрос (Query):** Вопрос к БД, сформулированный на специальном языке (например, SQL).
